# ZebraGate 上线须知

正式对外开放网站注册前，管理员必须完成以下配置。未完成时不要开放注册入口。

## 一、用户注册安全要求

### 1. 必须启用邮箱验证

管理员进入：

`系统设置 → 身份认证 → 基础认证`

开启“邮箱验证”，要求新用户注册时填写真实邮箱并通过验证码验证。禁止仅凭用户名和密码完成注册。

启用前必须先进入：

`系统设置 → 运维 → SMTP 邮件`

正确配置 SMTP 主机、端口、账号、发件地址和密码，并确认验证码邮件能够正常送达。SMTP 未配置成功时不得开启正式注册，以免用户无法收到验证码。

### 2. 必须启用机器人防护

管理员进入：

`系统设置 → 身份认证 → Bot Protection`

填写 Cloudflare Turnstile 的 Site Key 和 Secret Key，然后开启“启用 Turnstile”。该功能用于拦截机器人注册、脚本注册和批量注册。

不得只在前端显示验证组件；上线验收时必须确认缺少或伪造 Turnstile 凭证的注册请求会被后端拒绝。

### 3. 必须限制可用邮箱

建议在：

`系统设置 → 身份认证 → 基础认证`

开启“邮箱别名限制”，防止使用 `name+alias@example.com` 一类邮箱别名重复注册。

如业务只面向指定组织或客户，还必须开启“邮箱域名限制”，并配置允许注册的邮箱域名白名单。

### 4. 必须按需控制注册入口

“开放注册”和“密码注册”开关只应在邮箱验证、SMTP 和机器人防护均已配置并验收通过后开启。

遇到批量注册、验证码攻击或邮件服务异常时，应立即关闭“开放注册”，处理完成后再恢复。

## 二、上线前验收清单

- [ ] SMTP 配置正确，测试邮箱能够收到验证码。
- [ ] 注册页面必须填写邮箱和邮箱验证码。
- [ ] 错误、过期或已使用的邮箱验证码无法完成注册。
- [ ] 同一邮箱不能重复注册。
- [ ] Cloudflare Turnstile 已启用并能正常加载。
- [ ] 未完成人机验证的用户无法注册。
- [ ] 自动化脚本和高频批量注册会被拦截或限流。
- [ ] 邮箱别名限制已按业务需要开启。
- [ ] 邮箱域名白名单已按业务范围配置。
- [ ] 管理员已确认可以随时关闭开放注册。

以上项目全部通过后，才能正式开放用户注册。
