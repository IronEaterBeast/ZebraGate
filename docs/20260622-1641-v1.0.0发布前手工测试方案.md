# ZebraGate v1.0.0 发布前手工测试方案

## 1. 测试目标

确认 ZebraGate 云端管理系统与 Windows 桌面客户端在真实用户操作下可以完成安装、登录、模型分组、本地代理调用、隐私保护、托盘运行、开机自启动和自动更新，并确认异常情况下有明确反馈且不会损坏用户配置。

## 2. 测试环境

至少准备以下环境：

- 一台 Windows 10 22H2 电脑。
- 一台 Windows 11 最新稳定版电脑。
- 一个可正常登录且有可用额度、渠道和模型的测试账号。
- 一个无额度或无可用模型的测试账号。
- 已部署的测试环境云端服务及其正式域名、HTTPS 证书。
- 使用正式生产配置构建的 ZebraGate 安装包。
- 一个支持自定义 OpenAI 接口地址与密钥的图形化 AI 客户端，例如 Cherry Studio、Chatbox 或同类软件。
- 自动更新测试需要两个连续版本的已签名安装包，例如 `1.0.0-rc.1` 与 `1.0.0-rc.2`。

测试前记录安装包文件名、版本号、构建提交 ID、Windows 版本和测试环境地址。不要使用真实客户账号或生产密钥。

## 3. 通过标准

- 所有“高优先级”用例必须通过。
- 不得存在崩溃、数据丢失、无法退出、代理错误转发、隐私保护失效或更新后无法启动的问题。
- 中低优先级问题必须明确记录影响范围和处理结论。
- 修复问题后，需要复测原失败用例，并回归与其相关的完整功能链路。

## 4. 云端管理系统

### 4.1 服务与页面访问（高优先级）

1. 使用全新浏览器窗口访问测试环境首页。
2. 检查页面能正常加载，标题、Logo、产品名称均为 ZebraGate。
3. 刷新页面并直接访问登录、控制台等内部页面。
4. 检查不存在白屏、资源 404、明显控制台错误或旧品牌文案。

预期：页面和静态资源正常；刷新及直接访问内部地址不会丢失路由。

### 4.2 登录与会话（高优先级）

1. 输入错误密码，确认页面提示明确且不会登录。
2. 使用正确账号登录。
3. 刷新页面并关闭浏览器后重新打开。
4. 主动退出登录，再尝试访问需要登录的页面。

预期：正确登录后会话正常保持；退出后受保护页面不可继续访问；错误提示无敏感信息。

### 4.3 账号、额度与模型（高优先级）

1. 查看当前账号信息、额度和可用模型。
2. 在管理端调整测试账号的额度或可用模型范围。
3. 刷新管理端，并在桌面端重新登录或刷新模型列表。

预期：云端数据展示正确；桌面端最终能获得与账号权限一致的额度和模型目录。

## 5. 桌面客户端安装与启动

### 5.1 全新安装（高优先级）

1. 在未安装过 ZebraGate 的 Windows 电脑上运行安装包。
2. 按默认选项完成安装并启动。
3. 查看开始菜单、已安装应用列表和程序图标。

预期：安装过程无异常；应用名称和图标正确；首次启动不会弹出缺少组件或权限错误。

### 5.2 单实例（高优先级）

1. 保持 ZebraGate 已运行。
2. 再次双击桌面或开始菜单中的 ZebraGate。
3. 连续快速启动两到三次。

预期：始终只有一个进程实例和一个本地代理；重复启动只显示已有主窗口，不出现端口占用提示。

### 5.3 窗口与多显示器（中优先级）

1. 分别在 100%、125% 和 150% 缩放下打开主窗口和分组管理窗口。
2. 在双显示器环境中把任务栏放在不同屏幕或不同边缘。
3. 显示、隐藏并重新打开窗口。

预期：窗口完整可见、不超出工作区；按钮和文字不重叠；窗口位置符合当前显示器工作区。

## 6. 登录与账号切换

### 6.1 正常登录（高优先级）

1. 在桌面端点击登录。
2. 在打开的登录页面完成登录授权。
3. 返回桌面端查看账号、额度、模型和代理状态。
4. 退出并重新启动桌面端。

预期：授权后桌面端自动识别登录状态；重启后登录状态按设计恢复；凭据不会以明文显示。

### 6.2 取消及失败登录（高优先级）

1. 启动登录后关闭网页，不完成授权。
2. 使用错误凭据尝试登录。
3. 登录过程中暂时断开网络。

预期：桌面端保持未登录状态，可以重新发起登录；不会无限等待、崩溃或保存无效会话。

### 6.3 账号切换与数据隔离（高优先级）

1. 使用账号 A 创建分组并选择模型。
2. 退出账号 A，登录账号 B，创建不同配置。
3. 再次切换回账号 A。

预期：两个账号的分组、模型选择和相关状态互不串用；账号 A 的配置能够正确恢复。

## 7. 分组与模型管理

### 7.1 分组增删改（高优先级）

1. 创建多个分组，输入正常名称、较长名称和重复名称。
2. 重命名分组。
3. 删除普通分组、默认分组，并尝试删除最后一个分组。
4. 重启应用后再次检查。

预期：有效操作立即生效并持久化；不允许删除最后一个分组；无效输入有明确反馈。

### 7.2 模型选择（高优先级）

1. 为不同分组选择不同模型，并保存。
2. 清空某个分组的模型选择。
3. 重启应用，确认模型选择仍然正确。
4. 在云端移除一个已选模型，然后让桌面端刷新模型目录。

预期：选择结果按分组保存；空模型分组不能错误转发请求；失效模型被安全移除并给用户明确通知。

### 7.3 离线缓存（中优先级）

1. 在线登录并成功加载一次模型目录。
2. 退出应用，断开网络后重新启动。
3. 查看已有模型和错误提示。
4. 恢复网络并等待或触发刷新。

预期：离线时可以看到缓存数据；不会把临时网络错误误报成配置丢失；网络恢复后可以重新同步。

## 8. 本地代理

### 8.1 图形化客户端接入（高优先级）

1. 启动 ZebraGate 本地代理。
2. 将界面展示的本地接口地址和分组密钥填入图形化 AI 客户端。
3. 发送一条普通聊天消息。
4. 观察流式输出、最终回复、ZebraGate 状态和云端额度变化。

预期：请求成功并正常流式返回；客户端填写的模型会被当前分组模型正确替换；额度与使用记录符合预期。

### 8.2 多分组路由（高优先级）

1. 为两个分组配置不同模型和不同本地密钥。
2. 在图形化客户端分别使用两个密钥发送请求。
3. 检查云端实际使用的模型。

预期：每个密钥只使用对应分组的模型，不会跨组或复用其他账号的数据。

### 8.3 错误场景（高优先级）

依次验证以下场景：未登录、分组未选择模型、错误分组密钥、云端不可达、账号无额度、云端返回业务错误、本地端口被其他程序占用。

预期：每种场景都有可理解的错误反馈；不会静默失败、无限重试或转发到错误模型；关闭占用程序后可以恢复。

### 8.4 并发与长回复（中优先级）

1. 同时从两个客户端发送请求。
2. 发送一条会产生较长流式回复的请求。
3. 回复过程中隐藏和重新显示主窗口。
4. 回复过程中尝试退出应用。

预期：并发请求互不串流；窗口操作不影响代理；退出时明确说明对进行中请求的影响且不会残留进程。

## 9. 隐私保护

### 9.1 默认开启与本地拦截（高优先级）

1. 全新安装或清除测试配置后首次启动。
2. 确认隐私保护默认开启。
3. 发送包含已配置敏感关键词的用户消息。
4. 检查图形化客户端反馈和云端请求记录。

预期：请求在本地被拦截，提示来自本地化文案；云端没有收到该请求，也不扣减额度。

### 9.2 开关与角色范围（高优先级）

1. 关闭隐私保护后再次发送相同内容。
2. 重启客户端，确认开关状态被保存。
3. 验证普通安全内容不会被误拦截。
4. 验证助手回复或历史错误文本不会被当作新的用户敏感内容再次拦截。

预期：关闭后请求可正常转发；开关持久化；扫描范围符合设计，不产生明显误拦截。

## 10. 托盘、自启动与退出

### 10.1 托盘菜单（高优先级）

1. 隐藏主窗口，通过托盘重新显示。
2. 重复点击显示/隐藏。
3. 打开分组管理、检查更新等托盘入口。
4. 检查菜单文字与窗口实际状态是否一致。

预期：托盘入口均可用；不会重复创建窗口；菜单状态和文案正确。

### 10.2 开机自启动（高优先级）

1. 从托盘启用开机自启动。
2. 在 Windows 启动应用设置中确认状态。
3. 重启 Windows，观察 ZebraGate 是否自动运行。
4. 禁用开机自启动并再次重启验证。

预期：开关与系统设置一致；启用时仅启动一个实例；禁用后不再自动启动。

### 10.3 退出（高优先级）

1. 本地代理运行时从托盘选择退出。
2. 阅读确认提示后分别选择取消和确认。
3. 退出后从图形化客户端再次发送请求。

预期：取消后应用与代理继续工作；确认后窗口、托盘和代理全部退出，端口被释放，无残留进程。

## 11. 自动更新

### 11.1 已是最新版本（高优先级）

1. 安装更新清单中的最新版本。
2. 分别从主界面和托盘执行检查更新。

预期：明确提示已经是最新版本；不会重复下载或报签名错误。

### 11.2 正常升级（高优先级）

1. 安装较低版本，并准备指向较高版本的有效签名更新清单。
2. 执行检查更新，确认显示的新版本号正确。
3. 先取消一次，再次检查并确认更新。
4. 等待下载安装和重启。
5. 检查新版本号、登录状态、分组、模型选择、隐私开关和自启动状态。

预期：取消不会修改程序；确认后签名校验、安装和重启成功；用户配置不丢失；更新过程中不能重复启动第二个更新任务。

### 11.3 更新异常（高优先级）

依次验证更新清单不可访问、安装包下载中断、签名错误和磁盘空间不足。

预期：旧版本仍可正常使用；错误提示明确；不会安装未通过签名校验的文件，也不会导致程序无法再次启动。

## 12. 卸载与重装

1. 从 Windows 已安装应用中卸载 ZebraGate。
2. 确认程序、托盘、进程和自启动项均已移除。
3. 检查用户配置是否按产品设计保留或删除。
4. 重新安装并启动。

预期：卸载无残留运行项；重装可以正常启动；配置保留策略与产品说明一致。

## 13. 问题记录模板

每个失败项至少记录：

- 用例编号和标题。
- 测试环境、应用版本、构建提交 ID。
- 前置条件与实际操作步骤。
- 预期结果与实际结果。
- 截图、录屏或日志位置。
- 是否稳定复现以及复现次数。
- 严重程度：阻塞、高、中、低。
- 修复提交 ID、复测人和复测结果。

## 14. 发布前最终回归

问题修复完成后，使用最终安装包重新执行全部高优先级用例，并抽查中优先级用例。最终确认以下事项后方可发布：

- 全量自动测试、类型检查、i18n 检查和构建全部通过。
- Windows 10 与 Windows 11 的安装、启动、代理、退出流程通过。
- 正式 API 地址、Web 地址、更新地址与签名公钥均已替换为真实值。
- GitHub Release 所需 Secrets 已配置并经过测试版本验证。
- 版本号、更新清单、安装包和 Git 标签一致。
- CHANGELOG 内容与最终交付功能一致。
